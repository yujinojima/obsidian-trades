# obsidian-trades

A Karpathy-style LLM-maintained research wiki for a crypto algo-trading system.
This repo is a **public mirror of an Obsidian vault** — open the markdown files in any
viewer (GitHub renders them natively; Obsidian renders the wiki-link graph).

> **Status:** read-only public mirror, no live execution code in this repo.
> All trading happens upstream; this repo is a knowledge layer over that work.

## How to read this repo

- **Start at:** [`Training Journal/master.md`](Training%20Journal/master.md) for
  current system state, hypothesis, and active experiments.
- **For trade evidence:** [`wiki/synthesis/cross-strategy-trade-library.md`](wiki/synthesis/cross-strategy-trade-library.md)
  embeds 50 representative trade charts across 5 outcome buckets.
- **For per-prim interpretation:** [`Research/README.md`](Research/README.md) maps
  each top-5 prim to behavioural-finance and quant/physics references.
- **For prim chart galleries:** [`Research/galleries/README.md`](Research/galleries/README.md)
  shows 20 outcome-stratified trades per top-5 prim.

## Cadence

The vault is regenerated by Claude Code session by session. Daily logs accumulate; pattern
pages are extended (never reset); experiment pages move from `queued → running → done` as
backtests are run.

## Privacy / publishing

This is a **fully public** mirror. It contains:

- Trade outcomes (entry/exit/MFE/MAE/PnL %) — no dollar amounts attached to a wallet
- Strategy descriptions and code-design rationale
- Open research questions and active experiments
- Polymarket paper-trade intent (no live orders, no wallet identifiers)

It does NOT contain:

- Private keys, API secrets, passphrases, or seed phrases (verified by `scripts/build_galleries_and_relocate_charts.py` companion secret-scan; see commit messages for scan results)
- Wallet addresses (funder or EOA)
- Polymarket CLOB API key/secret/passphrase
- `.env` files or environment-bound config
- Any production database credentials or RPC URLs containing API keys
